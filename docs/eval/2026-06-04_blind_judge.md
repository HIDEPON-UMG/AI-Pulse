## 評価所感（Claude 中立分析・ジャッジコメント照合済）

### 総合は 35B > flash-lite > qwen3 だが、内訳を見ると「強みの交換」
- **factual（事実忠実度）**: flash-lite **3.90** > 35B 3.30 ≈ qwen3 3.20。**ローカルは固有名詞・数値の捏造がやや多い**（35B が "Rajesh Jha"/"郭松教授"/"Dell,Cisco"、qwen3 が "Prof. GUO Song"/前回調達額 を捏造）。ニュース DB では最重要の watch-item。
- **summary**: 35B 4.00 ≈ flash-lite 3.90 > qwen3 3.30。
- **points**: flash-lite 3.90 > 35B 3.70 > qwen3 2.80。
- **rationale**: 35B **4.20** >> qwen3 2.80 >> flash-lite **1.80**。

### 検証で判明した重要事項
1. **flash-lite の rationale 1.80 は捏造でなく正当**。ジャッジが 7/10 で「importance/impact/buzz が `high`/`mid` のラベル反復のみで説明が無い」と指摘。**同一サンプルでも初回 flash-vs-lite eval では full 文だった** → temperature 0.4 のばらつき or プロンプト改変が疑われ、**flash-lite が本番でも rationale をラベルだけで出している可能性**（=既存の品質課題）。要別途確認。
2. **35B は rationale を一貫して深く書く**（指示遵守が強い）。flash-lite/qwen3 は importance 軸をラベルに退化させがち＝**プロンプトの rationale 指示が弱く、小型モデルが落ちる**。35B はこれに頑健。
3. **qwen3 は強調無効条件でも markdown 自己言及をリーク**（"下線された"/"==マーカー==された"）。プロンプトに強調指示が残るため。→ **強調コード化は検証だけでなくプロンプトからも強調指示を除去**しないと qwen3 のリークは消えない。
4. **factual の捏造の一部は本文 3000 字打切り由来**（devin 系で全モデルが 89%/調達額を捏造）。`MAX_BODY_CHARS` 引上げで全モデル改善余地。
5. **強調無効条件で全モデル 11/11 完走**＝qwen3 の脱落は強調契約が原因と確定。**35B のレイテンシも ~17s/件に短縮**（前回 44s は emphasis リトライで膨れていた）。76 件直列で 35B≈22 分 / qwen3≈16 分。

### 結論（品質差分は許容範囲か）
- **35B-A3B ≈ flash-lite（実質タイ。rationale で勝ち・factual で負け）**。総合では 35B が上だが、差の主因 rationale は flash-lite が以前は得意だった軸 → **品質は概ね互角で、ローカル化は「品質向上」でなく「品質維持＋クォータ非依存/オフライン/essay 一本化」が動機**と理解するのが正確。
- **qwen3:14b は明確に 3 番手**（factual・points 低・markdown リーク）。速度差(16 vs 22 分)は夜間バッチでは僅少。
